# Hermes Project Context: QuantumCylinder

## Project

QuantumCylinder is a small Python research codebase for the 2026 Quantum Information Hackathon QML challenge.

Main goal:

- Win or place highly by submitting a reproducible small-scale solution.
- Problem 1/2 baselines are locked.
- Problem 3 is the current focus.

## Current State

- Problem 1: random-unitary scrambling baseline is implemented.
- Problem 2: Hamiltonian projected diffusion baseline is implemented.
- Problem 2(d): comparable-strength resource matching is implemented.
- Problem 3: continuous measurement-induced denoising is implemented as the current main candidate.
- Qiskit is used for circuit/resource validation, not as the only source of truth.

## Coding Rules

- Prefer existing repo patterns over new abstractions.
- Problem-specific code belongs in `src/quantum_cylinder/problem_<n>_<topic>.py`.
- Executable experiments belong in `scripts/`.
- Configs belong in `configs/`.
- Tests use `pytest`.
- Run `python -m pytest` after code changes.
- Do not commit generated `results/` artifacts unless explicitly requested.
- Do not commit raw/private PDFs, application forms, phone numbers, emails, signatures, or other PII.

## Problem 3 Rules

The main Problem 3 idea is continuous measurement-induced denoising:

- Attach a complement qubit.
- Evolve with the fixed Problem 2 Hamiltonian.
- Post-select on a continuous complement basis.
- Compare against the original diffused ensemble and best axis-only `Z/X/Y` projection.
- Exact `Z/X/Y` axis bases are excluded from the continuous candidate search.

Do not claim a result as the main result unless the adoption gate passes:

- MMD or Wasserstein improves.
- Continuous candidate scores sufficiently above best axis-only candidate.
- Diversity retention is acceptable.
- Mean post-selection probability is acceptable.

If the default run says `fallback_candidate` or `do_not_use_as_main`, treat it as a candidate/appendix result only.
