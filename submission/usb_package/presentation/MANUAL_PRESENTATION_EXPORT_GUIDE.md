# Manual Presentation Export Guide

## Purpose

Use this guide when creating the final PDF or PPT from the submitted
presentation material.

Do not create separate main-round and final-round decks. The deck must be one
same English file for both rounds.

## Required Route

1. Prepare one 15-minute-capable English deck.
2. Use the 5-minute core path for the main round.
3. Use the same deck expanded into the 15-minute path for the final round.
4. Use appendix slides for Q&A.
5. Submit this same file. Do not submit a separate 5-minute deck.

## Export Steps

1. Open the presentation source used to create
   `QuantumCylinder_presentation.pdf`.
2. Apply the latest slide text from `PRESENTATION_SLIDE_TEXT_EN.md`.
3. Check the route in `PRESENTATION_STORYBOARD_EN.md`.
4. Export the one 15-minute-capable English deck to PDF or PPT.
5. Open the exported file on another machine.
6. Verify fonts, equations, tables, figures, and page/slide order.
7. Submit the same exported file for both main and final rounds.

## Required Content To Confirm

- 5-minute core path is clearly marked.
- 15-minute expanded path is included.
- IBM Cloud validation is visible in the core Problem 3(b) slide.
- Appendix A9 contains IBM job IDs and the beta-basis table.
- Haar reference is visible in the Problem 1 slide.
- Fidelity equation renders correctly.
- Post-selection equations render correctly.
- No local absolute paths are visible.
- No IBM token, API key, CRN, or private credential is visible.
- No hardware advantage claim appears.

## IBM Claim Guardrail

IBM Cloud/QPU validation is hardware-execution validation only. It validates
tiny representative Problem 3(b) mechanism circuits and does not replace the
state-vector benchmark.

Do not claim hardware advantage, IBM superiority, quantum advantage, or a full
trainable QuDDPM.
